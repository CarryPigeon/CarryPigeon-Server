## 1. 启动前置条件

- `config/application.yaml` 中的 `cp.chat.auth.jwt.secret` 必须提供，且长度至少 32 位
- `config/application.yaml` 中的 `cp.chat.server.id` 建议显式区分，避免多个实例共用默认 serverId
- MySQL / Redis / MinIO 必须可连接
- 本地 Docker 依赖建议先通过 `docker-up` 启动并等待健康
- 分发包启动时，推荐先编辑分发包根目录 `config/application.yaml`
- 分发包构建后，推荐先执行一次 `verify`，交付到目标环境前再执行一次 `verify --strict-config`
- Windows 分发包既可通过 `.bat` 入口执行，也可直接执行同目录下的 `.ps1` 脚本

## 2. 推荐流程

1. 先启动外部依赖
2. 构建分发包
3. 先执行分发包校验
4. 若要交付或归档，再生成发布校验文件
5. 再启动应用
6. 若需要后台运行，使用后台启动脚本
