# 产品需求文档优先级待办清单

## 1. 文档目的

本文基于 `docs/产品需求文档覆盖矩阵.md`，把尚未完全覆盖的 PRD 后端能力转化为可执行的优先级待办清单，用于后续迭代排期。

## 2. 使用规则

- 只纳入**当前后端仓库**相关的待办项。
- 只从覆盖矩阵中的 `Partial`、`Not Implemented` 中提炼待办。
- `Out of Current Scope` 项不进入当前待办池。
- 排序原则：**缺失核心契约 > 对外可见能力补齐 > 后续增强能力**。

## 3. 当前结论

- 基于当前覆盖矩阵，原先纳入本清单的主链路缺口与关键增强缺口已全部完成。
- 当前剩余工作已不属于本清单定义时的“待补齐缺口”，而属于新的产品增强排期范围。

## 4. 使用建议

- 这份文档可作为当前阶段缺口已清空的记录保留，而不是继续充当新的增强需求池。
- 若后续继续推进增强项，应基于最新覆盖矩阵与新的产品目标重新形成新的排期文档，而不是直接复用本清单。
