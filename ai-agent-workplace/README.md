# AI Agent Workplace

## 1. 目录职责

该目录是项目中 AI 智能体的固定工作目录。

用于承接：

- 临时分析材料
- 任务拆解记录
- 中间生成文件
- 待确认方案草稿

## 2. 目录边界

- 这里不是正式业务源码目录
- 这里不是正式规范文档目录
- 这里不是长期存放正式实现代码的地方

正式规则请写入 `docs/`，正式代码请写入对应模块源码目录。

## 3. 使用原则

- AI 的中间产物优先放在这里
- 已确认的最终内容应及时迁移到正式目录
- 不要把根目录当作 AI 的临时工作区

## 4. 文件类型

本目录只承接 AI 工作流材料，常见类型包括：

- `*-current.md`：当前仍在执行或等待确认的任务单。
- `*-done.md`：已完成并记录验证、自检、残留风险的任务单。
- `archive/`：已完成的历史任务单、被正式文档承接的草稿、阶段性总结。
- 临时分析材料：仅在任务执行期间保留，完成后应删除、合并或归档。

## 5. 清理规则

- 根目录应只保留 `README.md`、当前任务单和少量仍需活跃处理的文件。
- 已完成任务单应移入 `archive/`，避免根目录长期堆积历史文件。
- 明显重复的总结壳、空文件、占位文件和无独立信息量草稿应删除。
- 被正式 `docs/` 承接的 PRD、设计草稿和规则草稿应归档，不继续充当正式依据。
- 不得删除唯一记录用户确认、关键决策、验收结果或复审结论的文件。

## 6. 与正式文档的关系

- 长期规则、统一流程和架构边界必须写入 `docs/`。
- 本目录可以保存任务级追溯材料，但不能替代正式规范。
- 若任务产生可复用经验或稳定质量门禁，应在任务单中记录是否已回写 `docs/`。
