## 1. 文档目的

本文用于定义 CarryPigeon 服务端内的聊天核心模型，包括用户、身份、成员、会话、频道、消息、在线状态和实时收发链路。

## 2. 设计边界

聊天内核只负责通用聊天能力，不承载具体垂直场景业务。

场景能力应通过插件实现，例如 MC 服务器状态、GitHub 信息、文章评分等。

当前项目只专注服务端。客户端 UI 与客户端插件不纳入本阶段服务端研发范围。
