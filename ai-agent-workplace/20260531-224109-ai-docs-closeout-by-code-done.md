任务名称：

按当前项目代码基线收口事务广播、discovery/realtime、一致性异常响应与 system channel 约束文档

任务目标：

以当前仓库正式代码为唯一基线，修正文档中与代码不一致的描述，收口以下四类问题的文档口径：

- 事务后广播 / 事务内广播的当前实现边界
- discovery / realtime 对外公开语义
- 异常响应规范与当前 HTTP 错误模型
- system channel 初始化与唯一约束现状

任务背景：

前一轮自检确认：

- 代码侧 message mention 广播已改为事务后发布，但 channel realtime 广播仍存在事务内发布点
- `/api/server` discovery 当前固定返回 `wsUrl` 与 capability 布尔值，不根据 realtime 开关动态降级
- 异常规范文档仍以 `CPResponse.code` 作为稳定对外约定，但代码实际采用 `HTTP status + ApiErrorResponse`
- system channel 已由 Flyway 迁移种子保证存在，但数据库层尚未建立唯一约束

- `docs/`
- `ai-agent-workplace/`

任务分解 / 执行计划：

1. 复核四类问题在代码、配置、迁移与现有文档中的真实落点。
2. 选择应收口的正式文档，并明确每份文档只写“当前代码事实”和“当前已确认边界”。
3. 修改正式文档，消除与代码基线的直接冲突。
4. 自检文档内容是否仍符合仓库架构与协作规范。
5. 记录实际结果、验证记录与残留风险，并关闭任务单。
