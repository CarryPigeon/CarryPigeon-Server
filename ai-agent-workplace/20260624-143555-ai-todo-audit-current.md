任务名称：
项目 TODO 任务审查与实施方案制定

任务目标：
在不直接修改正式代码的前提下，完成项目规范、模块边界与现有 TODO 标记的梳理，形成可供人工审核的 TODO 处理方案与后续实施边界。

任务背景：
用户要求先完整阅读相关文档和代码，理解项目后，再对项目中标记的 TODO 任务给出处理方案；用户明确要求先审核方案，再根据任务单执行。

只读探索 / 审查任务

- 默认不修改正式源码、正式测试、正式配置、正式文档

禁止修改范围：
- 不修改模块依赖方向
- 不修改架构文档与长期规则
- 不直接实施 TODO 重构

依赖限制：

任务分解 / 执行计划：
1. 阅读项目规范与协作规则，确认探索类任务边界。
2. 梳理模块结构、当前职责与公开能力。
3. 定位全部 TODO 标记并区分实现项、清理项、架构项、文档项。
4. 结合代码引用关系与现行文档，判断每个 TODO 的真实影响面。
5. 形成分阶段实施方案，明确需要用户先确认的边界。

关键假设与依赖：
- 当前以源码中的 `TODO` 注释为“标记的 TODO 任务”主范围。
- 若后续用户要求把“待办清单类文档”一并纳入，则需扩展任务范围并更新任务单。
- 架构、模块职责、装配位置、公开 API 暴露方式相关 TODO 默认视为需人工确认后再实施。

实现要求：
- 本任务只提供证据化分析与方案，不宣称 TODO 已完成。
- 结论必须引用仓库内文档或代码位置作为依据。

测试要求：

- 已明确本次无正式代码改动。
- 已说明未执行测试 / 构建的原因。

复审要求：

文档要求：
