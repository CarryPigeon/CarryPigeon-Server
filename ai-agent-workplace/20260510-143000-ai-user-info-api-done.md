# 用户信息 API 迭代任务单

## 任务目标
整理并推动用户信息相关 API 的迭代方案，重点澄清当前用户身份态与资料态的边界，并补齐资料不存在、字段分层与测试覆盖等问题。

## 影响模块
- `chat-domain`
- `infrastructure-service`
- `application-starter`
- `docs/`（仅在需要补充长期规则时）

## 允许修改范围
- 用户信息相关 controller / application service / domain repository / DTO / test
- 用户资料持久化适配与数据库契约
- 与用户信息 API 直接相关的文档说明

## 禁止边界
- 不调整整体分层架构
- 不引入新的外部依赖
- 不修改无关业务模块
- 不在本任务中扩展到完整用户中心重构

## 依据文档
- `docs/架构文档.md`
- `docs/包结构规范.md`
- `docs/依赖引入规范.md`
- `docs/API.md`
- `docs/异常与错误码规范.md`
- `docs/测试规范.md`
- `docs/AI协作开发规范.md`
- `docs/变更审核清单.md`
- `docs/任务单模板.md`

## 当前问题
- `auth/me` 与 `users/me` 的语义边界仍不够清晰
- 用户资料不存在时的行为缺少明确约定
- 身份信息、展示资料、可编辑资料的字段分层不够明确
- 用户信息相关测试覆盖仍可加强

## 验收标准
- 能清晰说明 `auth/me` 与 `users/me` 的职责边界
- 用户资料不存在时的行为有明确约定并被测试覆盖
- 用户信息字段按语义分层，避免重复暴露
- 相关接口测试覆盖成功、失败、空串、未认证、未找到等关键场景
- 变更说明能对应到实际文件与影响范围
